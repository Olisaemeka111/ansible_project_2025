ARCHITECTURE AT A GLANCE:

  Internet
      ↓ (HTTP/HTTPS: 80, 443)
  ┌──────────────────────┐
  │  Web Tier (10 srvs)  │  ← Internet-facing (public subnets)
  └──────────┬───────────┘
             ↓ (App: 8000-9000)
  ┌──────────────────────┐
  │  App Tier (10 srvs)  │  ← Application servers (private subnets)
  └──────────┬───────────┘
             ↓ (DB: 3306,5432,27017,6379)
  ┌──────────────────────┐
  │   DB Tier (10 srvs)  │  ← Databases (private subnets, DENY ALL egress)
  └──────────────────────┘

  Traffic Flow: Internet → Web → App → DB
  Blocked Flow: Any reverse direction or cross-tier (except →DB)

═════════════════════════════════════════════════════════════════════════════

SECURITY HIGHLIGHTS:

  ✓ 3 security groups (tight rules, least privilege)
  ✓ No direct access to App or DB from internet
  ✓ Web ↔ App only on ports 8000-9000
  ✓ App ↔ DB only on specific DB ports
  ✓ Database tier: NO outbound traffic (⛔ DENY ALL)
  ✓ SSH access: Bastion host only
  ✓ Distributed across 3 availability zones
